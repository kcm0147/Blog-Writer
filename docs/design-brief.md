# 디자이너 브리프 — 네이버 블로그 자동 작성기

- **작성일:** 2026-05-15 (v3)
- **수신:** UI/UX 디자이너
- **요청 산출물:** HTML/CSS **마크업까지 포함된 정적 시안**
- **참고:** [판다랭크 블로그 글쓰기 2.1](https://pandarank.net/) (스크린샷 별첨 가능)

---

## 1. 한 줄 요약

작성자의 기존 블로그 글 스타일을 학습해, 매장 사진 + 메모를 입력받아 네이버 블로그용 약 1500자 초안 + 제목 + 해시태그를 자동 생성하는 **데스크탑 앱**.

---

## 2. 어떤 형태의 프로그램인가 (디자인에 영향 큰 컨텍스트)

- **Electron 데스크탑 앱** (Mac/Windows 모두 지원)
- **로컬에서 동작** — 서버 없음, 사용자 PC에 모든 데이터 저장
- **단일 사용자용** — 로그인/회원가입/멀티 유저 개념 없음
- 사용자가 본인의 LLM API 키(Claude 또는 Gemini)를 입력해 사용
- **비개발자용** — 기술 용어는 최소화하고, 한 번의 설정 후엔 평소처럼 폼만 채워 쓸 수 있어야 함

---

## 3. 타겟 사용자

- 네이버 블로그를 운영하는 개인 인플루언서/리뷰어
- 맛집/카페/장소 리뷰 글을 자주 쓰는 사람
- AI 도구에 친숙하지 않으며, 폼에 채우고 버튼 누르는 정도가 익숙한 수준

---

## 4. 핵심 가치 / 디자인이 강조할 것

1. **"내 스타일을 학습"** — 일반 AI 도구와 다르게 *내가 쓴 글*을 분석해 내 말투로 씀
2. **사진을 AI가 직접 봄** — 사진 속 분위기/메뉴/인테리어가 글에 자연스럽게 녹아듦
3. **빠르게 한 편 완성** — 메모 + 사진 → 30초 안에 제목/본문/해시태그 한 세트
4. **네이버에 그대로 복붙 가능** — 사진 삽입 위치까지 `[사진1]` 마커로 안내

---

## 5. 화면 구성 (반드시 포함되어야 할 5개 화면)

### 5.1 첫 실행 / 설정 화면 (Settings)

처음 앱을 열거나, 우측 상단 톱니바퀴 클릭 시.

- **LLM 제공자 선택** — 라디오 또는 탭: `Claude` / `Gemini`
- **API 키 입력** — 비밀번호 마스킹 필드 + "키 발급 받는 법" 링크
- **키 유효성 검증** 버튼 ("연결 확인")
- (선택) **웹검색 사용 토글** — "최신 매장 정보를 자동으로 가져옵니다" 설명 캡션
- **저장** 버튼

> 디자인 메모: API 키는 브라우저의 비밀번호 입력 UX와 유사하게. 보안에 신경 썼다는 느낌을 시각적으로 전달.

### 5.2 스타일 관리 화면 (My Style)

사용자의 기존 블로그 글을 모아 학습시키는 곳.

- **글 목록** (테이블/카드 리스트)
  - 컬럼: 제목/라벨, 글자 수, 추가 날짜, 삭제 버튼
- **+ 새 글 추가** 버튼 → 모달 열림
  - 모달 안: 라벨 입력 + 본문 텍스트 영역 (멀티라인, 큼직하게) + 저장
- **"스타일 분석 시작"** 버튼 (CTA)
  - 권장: 20~30개. 5개 미만이면 경고 표시.
- **분석 결과 카드** (분석이 끝나면 나타남)
  - 말투 비율 (예: 해요체 80%, 합니다체 20%)
  - 평균 문장 길이
  - 자주 쓰는 표현/감탄사 태그
  - 이모지 사용 빈도 (없음/낮음/중간/높음)
  - 구조 패턴 요약 한 줄

### 5.3 글 작성 화면 (Compose) — **메인 화면**

판다랭크 "블로그 글쓰기"를 참고하되, 더 미니멀하게.

**왼쪽 — 입력 패널** (스크롤 가능):

- **글 타입** (드롭다운): 맛집 후기 / 카페 / 여행 / 기타
- **매장명** (필수, 텍스트)
- **매장 주소** (필수, 텍스트)
- **방문일** (날짜 선택, 선택사항)
- **글 제목** (선택 — placeholder: "비워두면 AI가 추천해드려요")
- **SEO 키워드** (태그 입력, 최대 5개)
- **사진 업로드** — 드래그앤드롭 영역 + 썸네일 그리드
  - 최대 10장, jpg/jpeg/png, 5MB 자동 리사이징 안내
  - 썸네일에 순번(1, 2, 3 ...) 오버레이 — 본문 `[사진1]` 마커와 매칭
  - 드래그로 순서 변경 가능
- **글자 수** (탭/세그먼트): 500 / 1000 / **1500 (기본 강조)** / 2000
- **말투** (탭): **내 스타일 (기본 강조)** / 해요 / 합니다 / 반말
- **강조 / 제외 사항** (멀티라인 텍스트)
  - placeholder 예시: "친구랑 오후 2시 방문. 직접 결제했다는 표현은 빼줘. 2030 타겟."
- **방문 메모** (멀티라인, 큼직하게)
  - placeholder: "이번 방문의 키워드, 감상, 디테일을 자유롭게 적어주세요. 자세할수록 좋아요."

**오른쪽 — 결과 패널** (생성 전: 빈 상태, 생성 후: 결과 표시):

- **빈 상태:** 일러스트 또는 가이드 텍스트 ("왼쪽에 정보를 채우고 '글 만들기'를 누르세요")
- **로딩 상태:** 단계 표시 — `사진 분석 중… → 글 작성 중… → 마무리`
- **결과 상태:**
  - **제목** (편집 가능한 한 줄 입력, 큰 글씨)
  - **본문** (편집 가능한 멀티라인, `[사진1]~[사진N]` 마커를 **컬러 칩**으로 시각화 — 흐름 안에 자연스럽게)
  - **해시태그** (태그 칩 목록, 각 칩 클릭 시 복사 / "전체 복사" 버튼)
  - **사진 매핑 표** — `사진1 = 입구.jpg` 같은 작은 표
  - **액션 버튼**: `본문 복사` / `다시 만들기` / `히스토리에 저장됨 표시`

**하단 (항상 보임):**
- **글 만들기** 버튼 (큰 CTA, 입력이 충분치 않으면 disable)

### 5.4 히스토리 화면 (History)

과거 생성한 글 목록.

- 카드/리스트: 매장명 + 글 타입 + 생성일 + 글자수
- 클릭 시 → 결과 패널과 동일한 화면으로 열림 (재편집/재복사 가능)
- 검색 / 삭제 기능

### 5.5 공통 — 네비게이션

좌측 사이드바 또는 상단 탭:
1. ✍️ **글 작성** (Compose) — 메인
2. 📚 **내 스타일** (My Style)
3. 🕓 **히스토리** (History)
4. ⚙️ **설정** (Settings) — 우측 상단 아이콘이어도 됨

## 6. 톤 & 무드

- **깔끔하고 신뢰감 있는** 도구 같은 느낌
- 노션/리니어/라온 같은 **여백 있고 타이포가 살아있는** 디자인 톤 선호
- **이미지가 메인 콘텐츠처럼** 잘 보이는 레이아웃 — 사진이 곧 글의 재료라는 메시지
- 컬러는 자유. 단, **네이버 그린(#03C75A)은 피해주세요** (네이버 공식과의 혼동 방지)
- 다크 모드는 V1에서 안 해도 됨 (라이트만)

---

## 7. 상태 변형 (각 화면별로 포함)

1. **빈 상태** — 아직 글이 없을 때, 키워드/사진 없을 때
2. **입력 중 / 유효성 에러** — 필수 필드 누락 등
3. **로딩** — 생성 중, 분석 중
4. **결과 표시**
5. **에러** — API 키 오류, 네트워크 에러 등

---

## 8. 기술적 제약 / 디자인 결정에 영향을 주는 것들

- **데스크탑 앱** — 창 크기는 사용자가 리사이즈 가능. 기본 1280x800 정도를 가정.
  - **반응형:** 데스크탑(>900px) + 노트북(700~900px) 정도까지만. 모바일 X.
- **Electron 렌더러는 일반 웹 페이지** — 모든 HTML/CSS/JS 기술 그대로 사용 가능
- **다크모드 시스템 자동 감지** 같은 건 V1에서 안 함
- **글 생성은 20~30초 걸림** — 로딩 UX가 매우 중요 (단계별 진행 표시)

---

## 9. 산출물 요청 (HTML/CSS 마크업까지)

**필수:**
1. **HTML/CSS 정적 시안** — 위 5개 화면을 모두 포함하는 정적 마크업
   - 우리는 이 마크업을 React/Vue/Vanilla(추후 결정) 컴포넌트로 래핑함
   - 클래스 네이밍은 BEM 또는 utility-first (Tailwind 스타일) 자유 선택
   - 인터랙션(드롭다운 열기, 모달 열기 등)은 JS 안 짜셔도 되고, 정적인 상태별 마크업으로 OK
2. **상태별 마크업** — 빈/로딩/결과/에러 등 주요 상태가 별도 HTML로 보이도록
3. **컴포넌트 가이드** (간단해도 됨) — 버튼, 입력, 태그, 모달 등 자주 쓰이는 요소 정리
4. **타이포/컬러/스페이싱 토큰** — CSS 변수 또는 SCSS 변수로 정리

**포맷:** 깃 저장소에 폴더 통째로 받을 예정.

```
design/
├── index.html          # 화면 목록 인덱스 (각 화면으로 링크)
├── pages/
│   ├── compose.html
│   ├── compose-loading.html
│   ├── compose-result.html
│   ├── my-style.html
│   ├── history.html
│   └── settings.html
├── components.html     # 공통 컴포넌트 가이드
└── assets/
    ├── css/
    │   ├── tokens.css   # 색상/타이포/간격 변수
    │   └── styles.css
    └── img/
```

## 10. 참고 — 판다랭크와의 차별

판다랭크는 좋은 레퍼런스이지만 우리는 다음을 더 강조하고 싶습니다.

| 항목 | 판다랭크 | 우리 |
|------|---------|------|
| "내 스타일 학습" | 멤버십 유료 기능 | **핵심 기능 / 기본** |
| 사진 분석 | 단순 첨부 | **AI가 사진을 보고 글에 반영** |
| 결과 편집 | 단순 텍스트 출력 | **마커가 시각적으로 보이는 편집기** |
| 형태 | 웹 서비스 | **데스크탑 앱 — 내 PC의 도구** |
| SEO 인사이트 | 표시 | V1에서는 생략 (단순함 우선) |

**판다랭크보다 더 미니멀, 더 "내 도구"라는 정서**가 우선입니다.
